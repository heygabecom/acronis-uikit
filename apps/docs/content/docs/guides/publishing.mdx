# Publishing Guide

This guide explains how to build and publish `@acronis-platform/shadcn-uikit` from the monorepo.

## Prerequisites

- **pnpm** — install globally: `npm install -g pnpm`
- **npm account** with publish permissions to the `@acronis-platform` scope
- **npm access token** for CI/CD publishing

## Quick Start

```bash
# Install dependencies
pnpm install

# Build the library
pnpm --filter ./packages/ui-legacy run build

# Verify the build
ls packages/ui-legacy/dist/
```

## Package Structure

The published package includes:

```text
@acronis-platform/shadcn-uikit/
├── dist/
│   ├── index.js              # Main ES module entry
│   ├── index.d.ts            # TypeScript declarations
│   ├── components/ui/        # Individual component modules
│   ├── icons/                # Icon components
│   ├── lib/                  # Utilities
│   └── styles/
│       ├── index.css         # Main stylesheet
│       └── themes/           # Per-theme CSS files
├── package.json
├── README.md
└── LICENSE
```

## Version Management

Follow [Semantic Versioning](https://semver.org/):

- **MAJOR** — breaking changes
- **MINOR** — new features, backwards compatible
- **PATCH** — bug fixes, backwards compatible

See the [Versioning guide](/docs/guides/versioning) for how to create releases using `standard-version`.

## Manual Publishing

```bash
# 1. Create a release (bumps version, updates CHANGELOG, creates git tag)
pnpm release

# 2. Build the library
pnpm --filter ./packages/ui-legacy run build

# 3. Dry run to verify what will be published
cd packages/ui-legacy && npm publish --dry-run

# 4. Publish
cd packages/ui-legacy && npm publish --access public

# 5. Push the release commit and tag
git push --follow-tags origin main
```

## Automated Publishing (CI/CD)

```yaml
name: Publish Package

on:
  push:
    tags:
      - 'v*'

jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 22
          registry-url: 'https://registry.npmjs.org'
          cache: 'pnpm'

      - name: Install dependencies
        run: pnpm install

      - name: Build library
        run: pnpm --filter ./packages/ui-legacy run build

      - name: Publish to npm
        run: cd packages/ui-legacy && npm publish --access public
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
```

## Pre-publish Checklist

- [ ] All tests pass (`pnpm --filter ./packages/ui-legacy test`)
- [ ] Build completes without errors
- [ ] TypeScript declarations generated correctly
- [ ] Version number bumped
- [ ] `CHANGELOG.md` updated
- [ ] All new exports added to `packages/ui-legacy/src/index.ts`

## Build Verification

```bash
cd packages/ui-legacy

# Check dist contents
ls -la dist/

# Check bundle sizes
npm pack --dry-run

# Verify main export works
node -e "console.log(Object.keys(require('./dist/index.js')))"
```

## Beta / Pre-release

```bash
# Create a beta version
npm version prerelease --preid=beta
# → 0.34.0-beta.0

# Publish with beta tag
cd packages/ui-legacy && npm publish --tag beta

# Install beta in consuming projects
npm install @acronis-platform/shadcn-uikit@beta
```

## Troubleshooting

### Build errors

```bash
cd packages/ui-legacy
rm -rf dist
pnpm run build
```

### Package not found after publishing

Wait a few minutes for the npm registry to propagate, then:

```bash
npm cache clean --force
npm install @acronis-platform/shadcn-uikit
```

### Wrong files published

Check the `files` field in `packages/ui-legacy/package.json` — only listed paths are included in the tarball.
